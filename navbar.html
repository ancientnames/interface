How I Added My Navigation Panel to a Jupyter Notebook

Jupyter notebooks can be exported to HTML files. These exported HTML files can be edited like any other HTML file. This can be useful as part of an elaborate prank (i.e. sending someone the HTML export of a Jupyter Notebook with really esoteric nonsense code in the cells but changing the output of the cells to be surreal/haunting/mysterious/hard-to-believe ) or just for utility (adding a navigation bar so the notebook can be incorporated as a webpage).

Still, you should probably be familiar with some basic HTML code before you do this. To illustrate (and to store for future reference), here is how I added the navigation panel on my Jupyter Notebooks:

Step 1: Export the Jupyter Notebook to HTML

With your Jupyter notebook open, go to "File" -> "Download As" -> "HTML (.html)" to download the Jupyter notebook as an HTML. It will then ask whether to save the notebook or open it. Open the exported HTML in your preferred web-browser. The URL/address of the exported HTML tells you the location of the file on your computer if you want to edit the file directly. Otherwise, just save it to a new location wherever you want to edit it.

Step 2: Adding the Navigation Panel

First, place the HTML for the navigation panel in the body of the Jupyter Notebook exported HTML file. Use word-search/ctrl+f in a text editor (i.e. Visual Studio Code) to search for the <body> tag. Paste the navigation bar code after the tag, at the start of the body. For example, my navigation code HTML is as follows:

<div class="sidenav">
<a href="life.html">life</a>
<a href="fables.html">fables</a>
<a href="philosophy.html">philosophy</a>
<a href="code.html">code</a>
<a href="index.html">about</a>
</div>

Then, right before the body tag, add any CSS that your navigation bar relies on. All my CSS is stored in a file so I just added the following code before the start of the body:

<link rel="stylesheet" href="style.css">

Step 3: Making the Navigation Panel Consistent

At this point, you've probably noticed that the exported HTML has a lot of code in it. Some of it is probably altering the appearance of your side-bar. In my case, the text in my side-bar appeared with a different font and different vertical spacing than it did on other web-pages. Here is how I modified the HTML code to fix this:

I searched the HTML file for the following text and deleted the lines bolded in red:

html {
font-family: sans-serif;
-ms-text-size-adjust: 100%;
-webkit-text-size-adjust: 100%;
}

Then I searched the HTML file for the following text and deleted the lines bolded in red:

body {
font-family: "Helvetica Neue", Helvetica, Arial, sans-serif;
font-size: 13px;
line-height: 1.42857143;
color: #000;
background-color: #fff;
}

In practice, search for these texts by copying one of these lines without any padding at the ends because subtle things might be different (e.g. just search for "line-height: 1.42857143;" instead of the whole block). Doing this returned the font and spacing of my navigation bar to what it looks like on the rest of the (non-Jupyter) webpages.

In principle, one might be able to infer that these are the lines that should be deleted from knowledge of HTML -- but the Jupyter exports tend to have thousands of lines and I don't know or care how any of them work. As a reuslt, I did not do this the first time I modified an html file. Instead I identified these as the culprits by deleting large swathes of the Jupyter notebook files and seeing if they fixed the navigation bar. If they did, I undid my deletion and tried deleting half of what I originally did... and then half of that if it still worked. In essence, performing a binary search on the HTML code until I reached the minimal number of lines I needed to delete to solve the problem.
